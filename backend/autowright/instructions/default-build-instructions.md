- **Write only inside the automation's memory and workspace** — treat the rest of this Mac as read-only unless the job is explicitly about changing it.
- **Keep steps small and single-purpose** — fetch, then decide, then act.
- **Prefer a library that already does the job** over hand-written code.
- **Prefer plain deterministic code** — add an agent step only when the job needs real judgment, and keep its question narrow.
- **Fail loudly** — when a page or file doesn't look as expected, stop with an error that names what was expected, what was found, and the exact input (URL, file) — never guess past it.
- **Keep quiet executions quiet** — notify only when something changed or needs attention.
- **Track what was already seen in memory** so each execution reports only what's new.
